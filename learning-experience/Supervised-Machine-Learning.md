### Supervised Machine Learning: Regression and Classification

#### 第一周：机器学习与单变量线性回归

##### 1. 机器学习基础概念

- **机器学习定义**：让计算机从数据中学习规律，无需显式编程。
- **监督学习**：有标签数据，预测结果（如房价预测、垃圾邮件分类）。
- **无监督学习**：无标签数据，发现隐藏结构（如客户分群、异常检测）。

##### 2. 单变量线性回归 (Univariate Linear Regression)

- **模型表示**：\( f_{w,b}(x) = wx + b \)
- **代价函数 (Cost Function)**：平方误差函数 \( J(w,b) = \frac{1}{2m} \sum (f_{w,b}(x^{(i)}) - y^{(i)})^2 \)
- **目标**：寻找 \( w, b \) 使 \( J(w,b) \) 最小。
- **梯度下降 (Gradient Descent)**：
  - 公式：\( w = w - \alpha \frac{\partial J}{\partial w} \)，\( b = b - \alpha \frac{\partial J}{\partial b} \)
  - **学习率 α**：控制步长，过大易震荡，过小收敛慢。
  - **可视化**：碗状曲面，梯度指向上升最快方向。

##### 3. Python 工具链

- Jupyter Notebook 运行环境
- 基础库：NumPy, Matplotlib

> **项目应用提示**：建立最简单的预测模型（如根据面积预测房价），验证梯度下降是否收敛。

#### 第二周：多变量线性回归与工程优化

##### 1. 多变量线性回归 (Multiple Linear Regression)

- **模型扩展**：\( f_{\vec{w},b}(\vec{x}) = w_1x_1 + w_2x_2 + ... + w_nx_n + b \)
- **向量化 (Vectorization)**：用矩阵运算替代 for 循环，大幅提升计算效率。
  - 代码对比：`np.dot(X, w) + b` vs 逐个元素相乘累加。

##### 2. 梯度下降的实用技巧

- **特征缩放 (Feature Scaling)**：
  - 目的：加速收敛，避免等高线过于狭长。
  - 方法：均值归一化 (Mean Normalization)、Z-score 标准化。
- **学习率诊断**：
  - 绘制 \( J(\vec{w},b) \) 随迭代次数的曲线。
  - 若曲线上升或震荡，需减小 α；若下降过慢，可尝试增大 α。

##### 3. 特征工程 (Feature Engineering)

- **多项式回归**：构造新特征如 \( x^2, x_1x_2, \sqrt{x} \) 来拟合曲线。
- **工具库**：Scikit-learn 的 `LinearRegression` 类简化实现。

> **项目应用提示**：处理现实表格数据时，务必先做**数据标准化**。若线性拟合不佳，考虑**增加高次项特征**（多项式回归）来提升复杂度。

#### 第三周：分类、逻辑回归与正则化

##### 1. 逻辑回归 (Logistic Regression) - 二元分类

- **模型输出**：Sigmoid 函数 \( g(z) = \frac{1}{1+e^{-z}} \)，输出概率值 (0, 1)。
- **决策边界 (Decision Boundary)**：\( z = \vec{w}\cdot\vec{x} + b = 0 \) 对应的直线或曲线。
- **代价函数**：交叉熵损失 (Cross-Entropy Loss)，专用于分类问题，凸函数确保全局最优。

##### 2. 过拟合问题 (Overfitting)

- **表现**：训练集误差极低，但测试集/新样本表现差（高方差）。
- **解决手段**：
  1. 收集更多数据
  2. 减少特征数量（特征选择）
  3. **正则化 (Regularization)**（核心方法）

##### 3. 正则化 (Regularization)

- **原理**：在代价函数末尾添加惩罚项，抑制参数 \( w_j \) 过大。
- **线性回归正则化**：\( J(w,b) + \frac{\lambda}{2m} \sum_{j=1}^n w_j^2 \)
- **逻辑回归正则化**：同上，保持梯度下降更新公式一致。
- **λ (正则化参数)**：平衡拟合精度与模型复杂度。λ=0 为无正则化，λ过大导致欠拟合 (Underfitting)。

> **项目应用提示**：面对**高维特征且样本较少**的数据时，务必开启 **L2 正则化**（逻辑回归中的参数 `C` 对应 λ 的倒数）。如果模型在训练集上完美但在验证集上惨不忍睹，说明过拟合了，应加大正则化强度。

### 🛠️ 应用于新项目的检查清单

当你开始一个 ML 项目时，可以对照以下步骤回顾课程知识：

1. **数据加载与探索**：用 Pandas 读取，Matplotlib 可视化分布（复习 Week 1 Jupyter Lab）。
2. **预处理**：
   - 分类变量转数值（课程中未深讲，但属于基础扩展）。
   - **特征缩放**（Week 2 重点）：先划分训练/测试集，再用训练集均值方差标准化测试集。
3. **模型选择**：
   - 目标连续 → 线性回归 (Week 2)。
   - 目标 0/1 → 逻辑回归 (Week 3)。
4. **训练与调试**：
   - 绘制 **代价函数下降曲线** 确认收敛 (Week 2)。
   - 观察训练集/验证集误差判断 **过拟合** (Week 3)。
5. **优化**：
   - 尝试多项式特征 (Week 2)。
   - 调整正则化参数 λ (Week 3)。
